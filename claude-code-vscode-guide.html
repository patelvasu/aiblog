Architecture

How the Pieces Connect

Understanding the data flow helps you debug issues and make the right configuration choices.

Your prompt → Claude Code interprets and plans → Ollama routes the request → Minimax M2.7 reasons and returns code → Claude Code writes files to your project → you see the result in the browser.

Installation

Step-by-Step Setup

01
Install VS Code

Download Visual Studio Code for your platform. This is your code editor — the place where your project files live and where Claude Code's results will appear.

Download VS Code ↗
02
Install Node.js

Claude Code is built with JavaScript and requires Node.js to run. Download the LTS version for your platform.

Download Node.js ↗
Windows (PowerShell)
# Verify installation
node --version
03
Install Ollama

Ollama is the bridge between Claude Code and the AI model. Download and install it — it runs quietly in the background on your PC.

Download Ollama ↗

Supports Windows, macOS, and Linux. Click "Download for Windows" and run the installer.

04
Create a Free Ollama Account & Sign In

Open the Ollama app → Settings → Sign In. This redirects you to the Ollama website. Create a free account (email only). This gives you access to cloud-hosted models like Minimax M2.7.

05
Install Claude Code

Two methods — pick either one:

Option A — npm (recommended):

Terminal / PowerShell
npm install -g @anthropic-ai/claude-code

Option B — from Claude's website: Go to claude.ai/code, copy the install command shown, paste into PowerShell and press Enter.

Verify the installation:

PowerShell
claude --version
# Expected output: 2.1.8 (or newer)
06
Install the Claude Code VS Code Extension

In VS Code, open the Extensions panel (left sidebar), search for Claude Code, and install the first result. This lets you use Claude Code directly inside VS Code without opening a separate terminal each time.

The Key Command

Launching Claude Code

Once everything is installed, a single command starts your AI coding session. Open a New Terminal in VS Code, navigate to your project folder, and run:

📋 Bookmark this command
ollama  launch  claude  --model  minimax-m2.7:cloud
ollama — starts the bridge app you installed
launch claude — auto-setup, no manual config
minimax-m2.7:cloud — the AI brain (runs on Ollama's servers)
VS Code Terminal
# In VS Code: Terminal → New Terminal
ollama launch claude --model minimax-m2.7:cloud

# You'll see:
# Launching Claude Code with Minimax M2.7...
# ✓ Ready. Type your prompt below.

Using Ollama as an API

Ollama exposes a REST API you can call directly. The base URL is http://localhost:11434.

Generate a completion
curl http://localhost:11434/api/generate -d '{"model": "minimax-m2.7:cloud", "prompt": "Hello!"}'
Chat completion
curl http://localhost:11434/api/chat -d '{"model": "minimax-m2.7:cloud", "messages": [{"role": "user", "content": "Hi"}]}'
List available models
curl http://localhost:11434/api/tags

Usage

Building Your First Project

Once Claude Code is running, type your request directly in the terminal prompt. The agent will ask permission before creating files — click Yes when prompted. It will then generate all code and deliver a working project.

⚡ After confirming file creation, Claude Code writes all the code autonomously. You don't touch a line of code — just review the result in your browser.

Example: Landing Page

The tutorial demo built a full landing page for "Code Unveil" — an AI & data science learning platform. Here's the prompt used:

Claude Code Prompt
Make a dark landing page for Code Unveil — a website that
teaches Data Science, AI, and Coding. It should look cool
with a dark background, modern design, bright blue and green
highlights. Include a top hero section with a catchy headline
and a sign-up button, show how many active users are on the
platform, add a footer, and add some smooth animations to
make it feel alive.

Result: a fully animated, multi-section HTML/CSS page with a hero, stat counters, course cards, and footer — zero manual coding.

Pro Tips

Writing Prompts That Work

Think of Claude Code like a junior developer — talented, but it needs clear direction. The more context you give, the better the result. Here's how developers get the best outputs:

✕ Vague prompt

No context, no direction. Results in generic or off-target code.

"Make a website"
✓ Detailed prompt

Includes purpose, features, style, and expected behaviour.

"Create a restaurant landing page for 'Sakura House' with a menu section, gallery, and booking form. Use a dark elegant theme with gold accents."
✕ Unclear bug report

No file path, no reproduction steps, no expected vs actual behaviour.

"Fix the bug"
"Something broke"
"It doesn't work"
✓ Precise bug report

Pinpoints the exact component, describes the exact issue and expected fix.

"The mobile menu stays open after clicking a nav link. It should close. Check /components/Navbar.jsx."

"Add a dark/light mode toggle in the top right of the navbar."
✕ Ambiguous feedback

Subjective terms leave too much room for interpretation.

"Improve the hero section"
✓ Concrete changes

Lists exact properties to modify: size, color, spacing, effect.

"In the hero section, increase the H1 font size to 48px, add a linear gradient to the text color, and add a dark overlay behind the text for better readability."
